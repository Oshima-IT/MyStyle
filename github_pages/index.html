機能説明

magic_button

おすすめアイテム

仕様: Firestore の `items` コレクションを JSON にエクスポートし、選択中の系統タグと照合します。

ロジック: タグ一致 + 人気スコアで並び替え。静的ビルドでは JSON 内の `popularity` 値を利用します。

用途: GitHub Pages 上でも本番 UI と同じ推薦体験を再現します。

cookie

スタイル保存

仕様: 選択した系統は localStorage に保存し、次回アクセス時に復元します。

ロジック: Flask 版のセッション更新 API の代わりに、静的環境ではクライアントのみで完結させています。

用途: ログインなしでも自分の好みを保持できるため、手軽にプレビューできます。

sync_saved_locally

キャッシュ連携

仕様: `instance/weather_cache.json` や `wiki_trend_cache.json` を `github_pages/data/` にコピーして参照します。

ロジック: ビルド時に既存キャッシュを転写するだけで、静的ページから同じデータを確認できます。

用途: 本番の定期バッチをそのまま利用し、運用コストを増やさず GitHub Pages を公開環境に追加できます。
